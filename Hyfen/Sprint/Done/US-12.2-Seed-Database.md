# [US-12.2] Geração de Dados de Demonstração (Seed)

**Épico:** 12 - Governança e Segurança da Plataforma

**User Story:**
> **Como** um Desenvolvedor/Administrador,
> **eu quero** executar um único comando que popule o banco de dados com um conjunto de propriedades, acomodações e imagens de exemplo,
> **para que** eu possa desenvolver e testar a plataforma com conteúdo realista e ter material para demonstrações.

### Critérios de Aceite:

1.  ✅ Um novo "management command" do Django deve ser criado (ex: `python manage.py seed_database`).
2.  ✅ O comando deve criar entre 10 a 15 `Properties` falsas, mas com dados realistas (usando a biblioteca `Faker`).
3.  ✅ Para cada `Property`, devem ser criadas de 2 a 5 `Accommodations` também com dados realistas.
4.  ✅ O script deve baixar imagens de alta qualidade (com licença de uso livre, de fontes como Unsplash ou Pexels) e associá-las às propriedades e acomodações.
5.  ✅ O comando deve ser idempotente, ou seja, se for executado novamente, ele deve primeiro apagar os dados de demonstração antigos antes de criar novos, para evitar duplicação.
6.  ✅ Ao final da execução, o sistema deve estar em um estado funcional e visualmente atraente para uma demonstração.
